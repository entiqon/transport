# Security Policy

## Supported Versions

Security updates are provided for the **most recent minor release**.

Because the project is currently **pre-v1**, security fixes may include
breaking changes if necessary to resolve critical vulnerabilities.

| Version        | Supported |
|----------------|-----------|
| Latest minor   | ✅         |
| Older releases | ❌         |

---

## Reporting a Vulnerability

If you discover a security vulnerability:

1. **Do not open a public issue.**
2. Report the issue privately via email to the maintainers.
3. Include:
    - reproduction steps
    - affected version
    - potential impact

Contact:

security@entiqon.dev

If you do not receive a response within a reasonable timeframe,
please follow up.

---

## Disclosure Policy

The project follows responsible disclosure:

- Critical vulnerabilities will be patched **as quickly as possible**
- A release containing the fix will be published
- Public disclosure will occur **after a patch is available**

Security and stability take priority over feature velocity.
